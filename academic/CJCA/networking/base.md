# Overview
- Uniform Resource Locator (URL) ：an URL (https://www.hackthebox.eu/example?floor=2&office=dev&employee=17) also specifies the "floor," "office," "mailbox" and the corresponding "employee"
- Fully Qualified Domain Name (FQDN)： an FQDN (www.hackthebox.eu) only specifies the address of the "building"。从根开始、一路写到主机名、结尾带一个点“.”的绝对路径，保证在全球 DNS 命名空间里唯一、无歧义。www.example.com. ← FQDN（结尾的“.”显式写出）。www.example.com  ← 日常写法，仍是 FQDN；解析器会自动在尾部补根

# VPN 类型
- Site-To-Site VPN：客户端和服务器都是网络设备，通常是路由器或防火墙，并共享整个网络范围。这种方法最常用于通过互联网连接公司网络，使多个地点能够像在本地一样通过互联网进行通信。
- Remote Access VPN：客户端的计算机创建一个虚拟接口，该接口的行为就像处于客户端的网络中一样。OpenVPN，它会创建一个 TUN 适配器，让我们能够访问实验室。在分析这些 VPN 时，需要考虑的一个重要方面是加入 VPN 时创建的路由表。如果 VPN 只为特定网络创建路由（例如：10.10.10.0/24），这称为分割隧道 VPN，意味着互联网连接不会通过 VPN 传出。
- SSL VPN：网页浏览器中进行的 VPN

# Proxy
代理是指设备或服务位于连接中间并充当中介。当中介mediator是关键信息时中间的设备必须能够检查流量的内容。如果没有作为中介的能力，该设备在技术上是一个网关，而不是代理。
## 透明/非透明代理
- 透明代理时，客户端不知道它的存在。透明代理拦截客户端到互联网的通信请求，并充当一个替代实例。
- 非透明代理，我们必须被告知它的存在。为此，我们以及我们要使用的软件会被提供一个特殊的代理配置，以确保到互联网的通信首先被发送到代理。如果没有这个配置，我们就无法通过代理进行通信。

# CIDR换算
1. 子网掩码：总共是32位二进制，所以/24是24个1（网络位，把位置占死），8个0（主机位，一个子网里总地址数，也就是一个网络下几台主机， 8个0就是2的8次幂个主机位）
2. 通过子网掩码拆解的Network Address网络地址的start boundery：通过IPv4的二进制强制转换比如192.168.12.160，CIDR分好子网掩码的主机位，都清0，比如/26中，前26位是11111111 11111111 11111111 11 后6位 000000。那么192.168.12.160的二进制最后160是10100000，所以被CIDR规则强制转换成后6位为0也就是10000000, 所以192.168.12.160/26属于网络地址ip是192.168.12.128（子网的起始地址，用来标识整个子网，不可分配，子网的起点边界）- 广播地址
3. 通过子网掩码拆解的Broadcast Address广播地址的end boundery: 跟start boundery，如果是/26然后广播地址就是把ipv4的二进制最后160是10100000，全部后6为转换为1，也就是10111111，192.168.12.160/26的广播地址是192.168.12.191，所以可用地址范围为192.168.12.129 - 192.168.12.190
4. 拆分当前子网掩码的一个网段，需要从主机位借位，比如拆成3个就要2的2次幂，4，所以要借两位1, 所以每个被拆分的子网段的网络位回多两个1，主机位变成3个0，也就是每个子网下的子网数量是2的3次幂，8个.

# IPv6
网络前缀用于标识网络、子网或地址范围。接口标识符由接口的 48 位 MAC 地址形成，并在此过程中转换为 64 位地址。默认的前缀长度是 /64。然而，其他常见的前缀有 /32、/48 和 /56。如果我们想使用自己的网络，我们从提供商那里获得的前缀会比 /64 短（例如 /56）。

在 RFC 5952 中，定义了上述的 IPv6 地址表示法：

- 所有字母字符总是以小写书写。
- 一个区块的所有前导零总是被省略。
- 一个或多个连续的 4 个零（十六进制）区块可以缩写为两个冒号（::）。
- 从左边开始只能执行一次将其缩写为两个冒号（::）。

# VLAN
- Better Organization：网络管理员可以根据任何共同属性对终端进行分组。
- Increased Security： 网络分段禁止未经授权的成员在其他网络中嗅探网络数据包。VLANs
- Simplified Administration：网络管理员无需担心终端的物理位置。
- Increased Performance：随着所有端点的广播流量减少，网络可以使用更多的带宽。
